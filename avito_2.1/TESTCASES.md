# Тест-кейсы

## POST /api/1/item

Тест 1.1: Успешное создание со всеми полями.
Ожидаемый результат: 200 OK, в ответе есть id, sellerId, name, price, statistics, createdAt.

Тест 1.2: Создание без price.
Ожидаемый результат: 200 OK, price = 0 или null.

Тест 1.3: Создание без statistics.
Ожидаемый результат: 200 OK, statistics = значения по умолчанию или null.

Тест 1.4: Создание с price: 0.
Ожидаемый результат: 200 OK, price = 0.

Тест 1.5: Создание со statistics.likes = 0.
Ожидаемый результат: 200 OK, likes = 0.

Тест 1.6: name со спецсимволами.
Ожидаемый результат: 200 OK, name в ответе совпадает.

Тест 1.7: name с кириллицей, латиницей и цифрами.
Ожидаемый результат: 200 OK.

Тест 1.8: Очень длинное name (1000 символов).
Ожидаемый результат: 200 OK (или 400, если есть ограничение).

Тест 1.9: Очень большое значение price.
Ожидаемый результат: 200 OK.

Тест 1.10: sellerID нижней границы диапазона (111111).
Ожидаемый результат: 200 OK.

Тест 1.11: sellerID верхней границы диапазона (999999).
Ожидаемый результат: 200 OK.

Тест 1.12: Пустое Body.
Ожидаемый результат: 400 Bad Request, сообщение об ошибке.

Тест 1.13: Нет sellerID.
Ожидаемый результат: 400 Bad Request.

Тест 1.14: Нет name.
Ожидаемый результат: 400 Bad Request.

Тест 1.15: sellerID = null.
Ожидаемый результат: 400 Bad Request.

Тест 1.16: name = null.
Ожидаемый результат: 400 Bad Request.

Тест 1.17: name = “”.
Ожидаемый результат: 400 Bad Request.

Тест 1.18: sellerID строка.
Ожидаемый результат: 400 Bad Request.

Тест 1.19: sellerID дробное число.
Ожидаемый результат: 400 Bad Request.

Тест 1.20: sellerID отрицательное.
Ожидаемый результат: 400 Bad Request.

Тест 1.21: sellerID вне диапазона (меньше).
Ожидаемый результат: 400.

Тест 1.22: sellerID вне диапазона (больше).
Ожидаемый результат: 400.

Тест 1.23: price строка.
Ожидаемый результат: 400.

Тест 1.24: price отрицательное.
Ожидаемый результат: 400.

Тест 1.25: statistics не объект.
Ожидаемый результат: 400.

Тест 1.26: statistics.likes строка.
Ожидаемый результат: 400.

Тест 1.27: statistics.viewCount отрицательное.
Ожидаемый результат: 400.

Тест 1.28: Невалидный JSON.
Ожидаемый результат: 400.

Тест 1.29: Неверный Content-Type.
Ожидаемый результат: 415 или 400.

Тест 1.30: Лишние поля в JSON.
Ожидаемый результат: 200 OK, лишнее игнорируется.


## GET /api/1/item/{id}

Тест 2.1: Получение существующего id.
Ожидаемый результат: 200 OK, Body: массив с одним объектом.

Тест 2.2: Проверка данных.
Ожидаемый результат: все поля совпадают с созданным объектом.

Тест 2.3: Несуществующий id.
Ожидаемый результат: 404 Not Found.

Тест 2.4: Неверный формат id.
Ожидаемый результат: 400 или 404.

Тест 2.5: Пустой id
Ожидаемый результат: 404 или 405.


## GET /api/1/{sellerID}/item

Тест 3.1: Существующий sellerID.
Ожидаемый результат: 200 OK, массив объявлений.

Тест 3.2: Проверка данных.
Ожидаемый результат: массив содержит созданные объявления этого продавца.

Тест 3.3: Продавец существует, объявлений нет.
Ожидаемый результат: 200 OK, пустой массив.

Тест 3.4: sellerID строка.
Ожидаемый результат: 400.

Тест 3.5: sellerID отрицательное.
Ожидаемый результат: 400 или 404.

Тест 3.6: sellerID дробное.
Ожидаемый результат: 400.

Тест 3.7: sellerID вне диапазона.
Ожидаемый результат: 400 или 200 + пустой массив.
